# 0001 使用 Vite 承载组件库展示框架

## 状态

Accepted

## 背景

展示框架需要承载八个组件库页面，并作为 skill 的示例项目保存在本地技能目录。项目当前阶段不需要服务端渲染、路由数据加载、API 路由或部署平台耦合。

## 决策

使用 Vite、React、TypeScript、Vitest 和 Testing Library 创建展示框架。

## 影响

- 项目启动与构建配置较少，适合作为 skill 内的示例项目。
- 后续页面实现不依赖特定全栈框架。
- 若后续需要 Next.js 专属能力，需要迁移展示框架或新增独立示例。

## 替代方案

- Next.js：功能覆盖更广，但对当前静态展示页面偏重。
- 纯 HTML：维护成本低，但无法体现 React 页面组件结构。
- Storybook：适合组件开发，但当前目标是页面级风格展示。
