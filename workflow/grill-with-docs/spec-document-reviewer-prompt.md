# 设计自审检查清单

用于设计文档的自审检查。格式兼容分派审查子代理使用。

**目的：** 验证设计完整、一致，为实现计划做好准备。

```
分派 subagent（审查类型）：
  description: "审查设计文档"
  prompt: |
    你是一名设计文档审查者。验证此设计是否完整并准备好进入计划阶段。

    **要审查的设计：** [设计内容]

    ## 检查内容

    | 类别 | 关注点 |
    |------|--------|
    | 完整性 | TODO、占位符、"待定"、不完整的部分 |
    | 一致性 | 内部矛盾、冲突的需求 |
    | 清晰性 | 需求歧义是否足以导致某人构建错误的东西 |
    | 范围 | 是否聚焦到足以生成单个计划——不覆盖多个独立子系统 |
    | YAGNI | 未被请求的功能、过度工程 |

    ## 校准

    **只标记会在实现计划过程中造成实际问题的问题。**
    缺失的部分、矛盾、或歧义到可以被两种方式解释的需求——这些是问题。微小的措辞改进、风格偏好、"某些部分不如其他部分详细"这些不是。

    除非存在可能导致有缺陷计划的严重缺口，否则批准。

    ## 输出格式

    ## 设计审查

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [第 X 部分]: [具体问题] - [为什么对计划重要]

    **建议（咨询性质，不阻止批准）：**
    - [改进建议]
```

**审查者返回：** 状态、问题（如有）、建议
