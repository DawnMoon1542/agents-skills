# 规格一致性审查者提示模板

在分派规格一致性审查 subagent 时使用此模板。

**目的：** 验证实现者构建了被要求的内容（不多不少）

**在代码质量审查之前分派。** 先确认实现了正确的东西，再检查实现方式是否正确。

```
分派 subagent（审查类型）：
  description: "审查任务 N 的规格一致性"
  prompt: |
    你正在审查一个实现是否匹配其规格说明。

    ## 被要求的内容

    [任务需求的完整文本]

    ## 实现者声称构建了什么

    [来自实现者的报告]

    ## 关键：不要相信报告

    实现者完成得可疑地快。他们的报告可能不完整、不准确或过于乐观。
    你必须独立验证一切。

    **不要：**
    - 相信他们对于实现了什么的说法
    - 相信他们关于完整性的声明
    - 接受他们对需求的解释

    **要：**
    - 阅读他们写的实际代码
    - 逐行对比实际实现与需求
    - 检查他们声称实现但实际缺失的部分
    - 寻找他们没有提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们实现了被要求的所有内容吗？
    - 是否有跳过或遗漏的需求？
    - 他们是否声称某些东西有效但实际上没有实现？

    **额外/不必要的工作：**
    - 他们构建了未被要求的东西吗？
    - 他们过度工程或添加了不必要的功能吗？
    - 他们添加了不在规格中的"锦上添花"吗？

    **误解：**
    - 他们是否以不同于预期的意图解释了需求？
    - 他们是否解决了错误的问题？
    - 他们是否以错误的方式实现了正确的功能？

    **通过阅读代码验证，不要相信报告。**

    报告：
    - 规格一致（如果代码检查后一切匹配）
    - 发现问题：[具体列出缺失或多余的内容，带文件:行号引用]
```

**审查者返回：** 规格一致或不一致的具体问题列表
