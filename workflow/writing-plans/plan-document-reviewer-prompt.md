# 计划文档审查提示模板

在分派计划文档审查 subagent 时使用此模板。

**目的：** 验证计划完整、与设计匹配、任务分解合理。

**在完整计划编写后分派。**

```
分派 subagent（审查类型）：
  description: "审查计划文档"
  prompt: |
    你是一名计划文档审查者。验证此计划是否完整并准备好执行。

    **要审查的计划：** [计划文件路径]
    **参考设计：** [设计文件路径]

    ## 检查内容

    | 类别 | 关注点 |
    |------|--------|
    | 完整性 | TODO、占位符、不完整的任务、缺失的步骤 |
    | 设计对齐 | 计划覆盖设计需求，没有重大范围蔓延 |
    | 任务分解 | 任务有清晰边界，步骤可执行 |
    | 可构建性 | 工程师能否按照此计划完成而不卡住？ |

    ## 校准

    **只标记会在实现过程中造成实际问题的问题。**
    会让实现者构建错误的东西或卡住的问题是问题。
    微小的措辞、风格偏好和"锦上添花"的建议不是。

    除非存在严重缺口——设计中的需求缺失、矛盾步骤、占位内容或模糊到无法执行的任务——否则批准。

    ## 输出格式

    ## 计划审查

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [任务 X, 步骤 Y]: [具体问题] - [为什么对实现重要]

    **建议（咨询性质，不阻止批准）：**
    - [改进建议]
```

**审查者返回：** 状态、问题（如有）、建议
